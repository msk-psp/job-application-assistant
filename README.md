# 입사 지원서 작성 도우미

이력서 프로필을 바탕으로 채용 지원서의 입력 항목을 찾아 값을 제안하고
채워 주는 로컬 우선 Chrome 확장 프로그램입니다. 한국어와 영어로 작성된
일반적인 항목명을 인식하며 사람인, 원티드, 리멤버, LinkedIn 및 일반 채용
페이지를 식별할 수 있습니다.

확장 프로그램은 지원서를 자동으로 제출하거나 파일을 업로드하지 않습니다.
비밀번호를 수집하지 않으며 이력서 데이터를 외부 서버로 전송하지 않습니다.
사용자가 확장 프로그램을 연 경우에만 현재 페이지에 일시적으로 접근하고,
백그라운드에서 사용자의 웹 탐색 활동을 추적하지 않습니다.

## 주요 기능

- 이력서 프로필을 `chrome.storage.local`에 로컬 저장
- 텍스트 기반 PDF 이력서에서 연락처, 링크, 기술 및 경력 텍스트 추출
- 추출 결과를 항목별로 검토하고 선택적으로 기존 프로필에 병합
- 회사, 직책, 재직 기간, 담당 업무, 성과 및 프로젝트를 구조화해 관리
- 검토한 서술형 질문과 답변을 분류·태그와 함께 보관하고 유사 질문에 추천
- 한국어 및 영어 지원서 항목명 매칭
- 사람인, 원티드, 리멤버, LinkedIn 전용 필드 어댑터와 일반 페이지 탐지
- 제안별 높은·보통·낮은 매칭 신뢰도 표시
- 입력 전 제안 내용 검토 및 항목별 선택
- React 기반 제어 입력 필드와 호환되는 이벤트 처리
- 사용자의 확인 없는 자동 제출 방지

## 개발 버전 설치

1. Chrome 주소창에서 `chrome://extensions`를 엽니다.
2. 오른쪽 위의 **개발자 모드**를 활성화합니다.
3. **압축해제된 확장 프로그램을 로드합니다**를 선택합니다.
4. 이 저장소의 디렉터리를 선택합니다.
5. 확장 프로그램 설정에서 이력서 프로필을 작성하고 저장합니다.
6. PDF 이력서를 사용하는 경우 **내용 추출**을 누르고 가져올 항목을 검토합니다.
7. 자주 사용하는 서술형 답변은 실제 질문, 분류, 태그와 함께 답변 보관함에 저장합니다.
8. 채용 지원 페이지에서 확장 프로그램을 연 뒤 **Scan form**을 선택합니다.
9. 신뢰도와 제안 값을 검토하고 필요한 항목만 선택한 뒤 **Fill selected**를 누릅니다.
10. 실제 제출 전 페이지에 입력된 내용을 직접 확인합니다.

Chrome 내부 페이지에서는 콘텐츠 스크립트를 실행할 수 없습니다. 확장 프로그램을
설치하기 전에 열어 둔 채용 페이지에서는 접근이 실패할 수 있으므로 해당 페이지를
새로고침한 뒤 다시 시도합니다.

## 개인정보 처리 원칙

- 이력서 프로필은 현재 Chrome 프로필 내부에만 저장됩니다.
- PDF 원본은 메모리에서만 처리하며 파일 자체를 저장하지 않습니다.
- 현재 버전에는 외부 API 호출이나 분석 수집 기능이 없습니다.
- 확장 프로그램 버튼을 누른 현재 탭에만 `activeTab` 권한을 사용합니다.
- 저장소에 실제 이력서, 연락처 또는 지원 답변을 커밋하지 않습니다.

공용 또는 다른 사람이 사용하는 컴퓨터에서는 민감한 프로필을 저장하지 마십시오.
Chrome 동기화 및 운영체제 계정의 보안 설정도 사용자가 직접 관리해야 합니다.

## 현재 지원 범위

현재 버전은 사용자가 확장 프로그램 설정에 직접 저장하거나 PDF에서 검토 후
가져온 구조화 프로필과 플랫폼별·일반 폼 항목 매칭을 사용합니다. PDF 가져오기는 텍스트가
포함된 15MB 이하 파일을 지원하며, 이미지로 스캔된 PDF와 암호화된 PDF는
지원하지 않습니다. 답변 추천은 사용자가 미리 검토해 저장한 답변만 사용하며,
외부 AI로 새로운 답변을 생성하지 않습니다. 각 채용 플랫폼의 이력서 페이지에서
정보를 자동으로 가져오는 기능은 아직 포함하지 않습니다.

사이트가 UI를 변경하면 플랫폼 어댑터의 라벨과 컨테이너 규칙도 업데이트해야 합니다.
낮은 신뢰도 제안은 기본 선택되지 않으며 모든 값은 입력 전에 사용자가 검토합니다.

## 테스트

Node.js 20 이상에서 다음 명령을 실행합니다.

```bash
npm run check
```

PDF 섹션 파싱, 구조화 경력 우선순위, 답변 추천, 신뢰도 분류 및 개인정보를 제거한
플랫폼별 폼 fixture를 테스트합니다. GitHub Actions에서도 Node.js 24로 동일한
테스트와 Manifest JSON 검사를 실행합니다.

## 개발 원칙

- 지원서 자동 제출 기능은 추가하지 않습니다.
- 사용자가 검토하지 않은 서술형 답변을 입력하지 않습니다.
- 플랫폼의 비공개 API나 인증 정보를 우회하여 사용하지 않습니다.
- 사이트 구조가 변경되면 플랫폼별 필드 매칭을 명시적으로 업데이트합니다.

## 포함된 오픈소스

PDF 텍스트 추출에는 Mozilla의 [PDF.js](https://github.com/mozilla/pdf.js)
`6.1.200` 배포본을 사용합니다. 라이선스 전문은 `vendor/pdfjs/LICENSE`에
포함되어 있습니다.
